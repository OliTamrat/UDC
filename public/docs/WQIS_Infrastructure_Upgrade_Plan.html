AI Cost Analysis: Gemini 2.5 Flash

The WQIS AI Research Assistant uses Google Gemini 2.5 Flash via the Google AI Developer API. This section provides exact cost calculations based on verified pricing.

Model & Pricing

ModelInput ($/MTok)Output ($/MTok)Status
Gemini 2.5 Flash$0.30$2.50In Use

Source: ai.google.dev/gemini-api/docs/pricing (Standard tier, verified June 27, 2026).

Gemini 2.5 Flash output pricing includes thinking tokens.
Gemini 2.5 Flash is a reasoning model. The $2.50/MTok output rate covers both the visible response and internal reasoning tokens. The per-query estimates below include a conservative thinking token overhead of ~50% of visible output, based on the domain-specific nature of water quality Q&A.

System Prompt

The AI Research Assistant sends a ~3,848-token system prompt with every API round-trip. This prompt includes: 12 monitoring stations, 25 water quality parameters with EPA thresholds, seasonal patterns, CSO/environmental justice context, and UDC research focus areas. Tool-use queries (fetching live station data) involve 2–3 round-trips, meaning the system prompt is billed multiple times per user question.

At current usage (0–5 researchers), AI costs just $23/year. At pilot scale (100 students), $476/year. AI only becomes the dominant cost driver at district scale (500+ students).

Cost Control Strategies (for future K-12 scaling)

  1. Rate-limit students to 5–10 queries/day — already implemented via checkRateLimit() in the chat API
  2. Cache frequent questions (e.g., "what is pH?", "what is E. coli?") to avoid API calls entirely
  3. Context caching — Gemini offers cached prompt pricing at $0.03/MTok input (90% discount). The 3,848-token system prompt is identical across all requests, making it an ideal caching candidate. Estimated savings: 30–40% on input costs.
  4. Restrict AI to teacher/researcher accounts; students use pre-built stories and scenarios instead
  5. Fund AI costs from grant technology line items at scale

Phase C: Sovereign Deployment Plan

Self-hosted deployment package for UDC to run WQIS on their own Azure subscription.

Delivery Model

DAPS Analytics builds and publishes a pre-built Docker image to our private Azure Container Registry (udcwqisacr). UDC receives a scoped pull token (read-only) that serves as a license key. Token renewal = subscription renewal. Source code never leaves DAPS.
